# 大赛简介

能源与气候危机愈演愈烈之下，可持续发展成为全球共识。互联网自然不能置身事外。技术人，如何用手中的一行行代码，守护 “绿色” 星球，让人类社会更加美好。充分利用 Serverless（无服务器）是减少能源消耗、践行低碳绝佳的方式之一。

Serverless 是一种云原生开发模型，能够让应用在远程托管的执行环境中按需运行，无需维护操作系统，从而提高资源利用率、降低成本、加速创新。

2022 年 3 月，Gartner 发布报告指出，到 2026 年，超过 50% 的全球企业将部署无服务器功能。不难看出，Serverless 的出现带来了一种全新的、颠覆式的云开发架构模式。为了践行 “技术向善” 的理念，帮助更多的开发者了解、熟悉无服务器技术，推动无服务器架构的创新，亚马逊云科技主办本次 Serverless for Good-2022 编程挑战赛。

# 赛程安排

  • 2022/7/6-2022/8/19：报名 & 初赛阶段
    • 提交内容：作品代码、技术文档
    • 评分指标：作品完成度 50%、技术架构 20%、使用体验 20%、创意度 10%
  • 2022/8/20-2022/9/18：决赛 & 答辩准备阶段
    • 提交内容：测试版作品（DEMO）、作品代码、技术文档、作品演示 PPT
    • 评分指标：作品完成度 30%、商业价值 30%、创意度 20%、使用体验 10%、技术难度 10%
  • 2022/9/23：决赛路演
    • 作品演示 PPT 最终版
    • 评分指标：作品完成度 30%、商业价值 30%、创意度 20%、使用体验 10%、技术难度 10%

# 初赛赛题

参赛选手从给定的两个场景中任选其一，基于 Amazon Serverless 架构实现。每个场景都给出了推荐架构，供参赛者参考，组委会在初赛前将安排一次统一的在线培训。开发者团队可采用推荐架构，也可自由进行架构设计。评委根据作品完成度、技术架构、创意度等评分标准进行评分，前 5-8 名团队进入决赛。

# 决赛赛题

参赛者基于 Amazon Serverless 架构，利用 Amazon Serverless 服务，实现社交泛娱乐、互动游戏、电商平台、IoT 等现代化应用场景的搭建，并实现交互。

你可以将 Serverless 架构用于物联网场景消除现实与数字世界之间的鸿沟，你可以考虑用 Serverless 解决社交场景下图像、视频处理等诸多问题，亦或是考虑开发一个电商网站，优化剁手党的购物体验… 更多场景等待着你去解锁！尽情发动你的想象力与创造力吧～

决赛阶段参赛团队必须完成能够试用的 Demo，可为 Web 网站、APP、小程序等，形式不限，使用的开发语言不限。本阶段将以 “线下编程 + 线上提交 + 现场答辩” 的方式进行。

# 作品提交

亚马逊云科技 Serverless 大赛采用线上提交作品的方式，提交方式请查看石墨文档《Serverless for Good-2022 选手作品提交方式》。
